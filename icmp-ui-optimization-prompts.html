L 布局结构优化

P01 三栏布局 → 主从式布局重构
Layout

提示词

请将当前 ICMP 分析工具的三栏布局（左侧控制 + 中间表格 + 右侧详情）重构为主从式布局：左侧保留 240px 固定宽度的控制面板，右侧主内容区分为上下两部分——上方是数据表格列表（占 55% 高度），下方是选中报文的字段解析详情面板（占 45% 高度）。表格行点击后，下方详情面板平滑展开并高亮对应行。要求：控制面板与主内容区之间用 0.5px 分割线，主内容区内部上下两区用可拖拽的分隔条调整比例。整体保持 8px 间距节奏，圆角统一 8px。
解决什么问题：当前三栏并列导致每栏空间局促，尤其右侧详情面板内容被压缩。主从式布局让数据列表获得更多横向空间，详情面板在需要时才展开。
P02 左侧面板分组折叠 + 渐进式披露
Layout

提示词

请将左侧控制面板改为渐进式披露结构：第一层始终展示「数据源选择」（离线文件/实时抓包/测试样本三个分段按钮 + 对应输入框），第二层「抓包参数」默认折叠收起，点击标题栏展开显示抓包数量、超时时间、过滤条件。每个折叠区使用 200ms 高度过渡动画，展开时标题栏箭头从 ▶ 旋转到 ▼。面板底部固定「开始分析」主按钮 + 「清空」「生成报告」次按钮，按钮上方显示实时状态指示器（绿色圆点 + 文字「就绪/分析中/完成」）。
解决什么问题：左侧面板所有控件平铺导致信息过载。折叠次要参数让用户聚焦核心操作，降低首次使用认知负担。

V 视觉层级优化

P03 建立 Design Token + 统一色彩语义
Visual

提示词

请为 ICMP 分析工具建立完整的 Design Token 系统：主色用 Blue #378ADD（信息/操作），语义色——Success #639922（就绪/校验通过）、Warning #BA7517（运行中/注意）、Danger #E24B4A（错误/不可达）、Purple #7F77DD（详情/高亮）。灰阶 7 级从 #F8F9FA 到 #2C2C2A。所有颜色以 CSS 变量定义，命名格式 --color-{name}-{level}。字体使用系统无衬线字体，字号阶梯 10/11/12/13/14/18px，仅用 400 和 500 两个字重。圆角统一 6px（输入框）/ 8px（卡片）/ 12px（容器），间距以 4px 为基准单位。所有边框使用 0.5px。
解决什么问题：当前界面蓝/绿/橙三色混用无规范，缺少语义色彩体系。统一 Token 后所有组件颜色有据可依，保证视觉一致性。
P04 按钮权重分层 + Tab 样式统一
Visual

提示词

请重新定义按钮和导航的视觉层级：主操作按钮（开始分析）用主色实心填充 #378ADD + 白色文字 + 36px 高度；次操作按钮（清空/生成报告）用 0.5px 线框 + 灰色文字 + 32px 高度；工具栏按钮（复制/导出）更小，28px 高度 + 图标+文字组合。顶部 Tab 导航从胶囊式改为下划线式——选中项底部 2px 主色线条，文字变为主色；未选中项灰色文字无边框。详情面板的子 Tab 用紫色系下划线区分层级。所有 hover 状态仅做颜色微调（加深 10%），不使用位移或阴影变化。
解决什么问题：当前所有按钮视觉权重相同，用户无法一眼识别主要操作。分层后操作优先级清晰，减少误操作。

I 交互体验优化

P05 表格行 ↔ 详情面板联动交互
Interaction

提示词

请实现数据表格与详情面板的联动交互：点击表格任意行后，该行背景变为 #E6F1FB 高亮，同时在下方详情面板自动展示该报文的字段解析（Ethernet/IP Header/ICMP Header 分组展示）。详情面板顶部有三个子 Tab（字段解析/原始数据/统计信息），默认选中「字段解析」。切换 Tab 时内容区用 150ms 淡入淡出过渡。键盘支持：↑↓ 方向键在表格行间移动并同步更新详情面板，Enter 键展开原始 Hex 数据，Esc 键取消选中。
解决什么问题：当前右侧详情面板与表格割裂，用户需手动操作才能查看详情。联动后查看效率提升 50% 以上。
P06 拖拽文件 + 快捷操作链路
Interaction

提示词

请增加拖拽上传和快捷操作：用户可直接将 .pcap 文件拖拽到主内容区，拖入时整个内容区显示半透明蓝色蒙层 + 中心大图标 + 「松开以加载文件」提示文字。文件加载后自动填充路径并启用「开始分析」按钮。增加键盘快捷键：Ctrl+O 打开文件、Ctrl+R 开始分析、Ctrl+L 清空结果、Ctrl+E 导出报告。快捷键提示在按钮 hover 时以 tooltip 形式显示。所有操作完成后在底部状态栏用 toast 消息反馈（3 秒自动消失）。
解决什么问题：当前操作路径较长（点浏览→选文件→点开始）。拖拽 + 快捷键让高频用户操作效率翻倍。

A 无障碍优化

P07 WCAG AA 合规 + 键盘全可达
A11y

提示词

请确保 ICMP 工具界面符合 WCAG 2.1 AA 标准：所有文字与背景对比度 ≥ 4.5:1（正常文字）/ 3:1（大文字），使用 #2C2C2A 作为正文色保证最大对比度。所有交互元素添加 ARIA 标签——Segmented Control 用 role="tablist" + aria-selected，表格用 role="grid" + aria-rowindex，详情面板用 aria-live="polite" 动态播报。键盘 Tab 顺序从左到右、从上到下，焦点环用 2px solid #378ADD + 2px offset 清晰可见。所有图标按钮提供 aria-label 文字描述。支持 prefers-reduced-motion 媒体查询关闭动画。
解决什么问题：当前界面缺少无障碍考虑，键盘用户和屏幕阅读器用户无法有效操作。合规后覆盖更广用户群。

S 状态设计优化

P08 空状态 + 加载态 + 错误态全覆盖
State

提示词

请为 ICMP 工具设计完整的 4 种状态界面：① 空状态——中央显示虚线圆形图标 + 「暂无解析数据」标题 + 「请选择 pcap 文件或开始实时抓包」描述 + 「选择文件开始」CTA 按钮；② 加载态——主按钮变为旋转图标 + 「分析中...」文字，状态指示器变为橙色脉冲圆点，表格区显示 3 行骨架屏（灰色色块 + 闪烁动画）；③ 错误态——红色叹号图标 + 错误标题 + 具体错误信息 + 「重试」按钮，背景用 #FCEBEB 浅红；④ 完成态——绿色对勾 + 「解析完成，共 N 条报文」+ 耗时统计。所有状态切换用 300ms 淡入淡出。
解决什么问题：当前空状态仅一行灰字「无数据」，加载和错误状态缺失。完整状态设计让用户始终知道「发生了什么、该做什么」。

D 数据展示优化

P09 表格增强 + 字段颜色编码
Data

提示词

请优化数据表格展示：表头固定不滚动（sticky header），表体区域滚动。每行支持 hover 高亮（#F8F9FA）和选中高亮（#E6F1FB）。ICMP 类型列用彩色 Badge 标签——Destination Unreachable 用红色底、Time Exceeded 用橙色底、Echo Reply 用绿色底、Echo Request 用蓝色底，文字白色或深色保证对比度。IP 地址和校验和列用等宽字体显示。新增「分类」列显示中文简称（不可达/超时/响应/请求）。表格右上角增加搜索框，支持按类型/IP 实时过滤。行数超过 100 时启用虚拟滚动。
解决什么问题：原始表格纯文字难以快速区分报文类型。颜色编码 + Badge 让分类信息一目了然，搜索过滤提升大数据量下的查找效率。
P10 字段解析树形展示 + 折叠分组
Data

提示词

请将详情面板的字段解析改为树形折叠结构：一级节点为协议层（Ethernet II / IP Header / ICMP Header / Payload），每个节点左侧有 ▶/▼ 折叠图标，默认展开全部。二级节点为具体字段，以「字段名：值」形式展示，字段名灰色 #5F5E5A 左对齐 120px 宽，值用等宽字体黑色显示。关键字段用语义色高亮——校验和正确用绿色、类型为错误类用红色、TTL 值过低用橙色。每个字段值右侧有「复制」小图标，点击复制单个值到剪贴板。Payload 区域用 Hex Dump 格式展示（左侧 Hex + 右侧 ASCII）。
解决什么问题：当前字段列表平铺无层次，大量字段同时展示信息过载。树形分组让用户按需展开，聚焦关心的协议层。

R 响应式适配

P11 宽屏优化 + 窄屏降级策略
Responsive

提示词

请为 ICMP 工具设计响应式适配方案：≥ 1200px 宽屏下，左侧面板 240px + 右侧主内容区自适应，表格显示全部 8 列，详情面板与表格上下分屏。900-1199px 中等屏宽下，左侧面板收窄至 200px，表格隐藏「分类」列，详情面板改为可关闭的底部抽屉。≤ 899px 窄屏下，左侧面板变为顶部可折叠横条（点击展开下拉），表格仅显示序号/类型/源IP 3 列，详情面板变为全屏覆盖式 Modal。所有断点切换用 300ms 过渡动画，不支持触摸手势操作（桌面工具优先）。
解决什么问题：当前界面固定尺寸无法适配不同屏幕。响应式方案确保在笔记本到大显示器上都有合理布局。

Dk 暗色模式优化

P12 完整暗色主题 + 自动跟随系统
Visual

提示词

请为 ICMP 工具设计完整的暗色主题：背景色用 #1A1A18（最深）/ #242420（面板）/ #333330（卡片），不用纯黑避免对比度过强。文字色反转——正文 #D3D1C7、次要 #888780、占位 #6B6966。主色 Blue 提亮至 #85B7EB 保证暗底可见性，语义色同步调整——Success 用 #97C459、Warning 用 #EF9F27、Danger 用 #F09595。所有边框改为 0.5px solid rgba(255,255,255,0.1)。表格 hover 用 rgba(255,255,255,0.05)，选中行用 rgba(133,183,235,0.15)。支持三种模式：亮色/暗色/跟随系统（通过 prefers-color-scheme 媒体查询自动切换），右上角下拉切换并记忆到 localStorage。
解决什么问题：网络分析工具常在低光环境下使用，暗色模式减轻视觉疲劳。完整的 Token 映射确保暗色下所有组件对比度达标。

M 微交互优化

P13 过渡动画 + 反馈微交互体系
Interaction

提示词

请为 ICMP 工具添加统一的微交互体系：① 按钮点击——150ms scale(0.97) 回弹反馈；② Tab 切换——下划线用 200ms slide 动画平滑移动而非闪烁出现；③ 表格行选中——背景色 150ms 渐变 + 左侧出现 2px 主色竖条；④ 折叠面板展开——250ms max-height + opacity 过渡；⑤ 数据加载完成——表格行从上到下依次 50ms 延迟淡入（stagger animation）；⑥ 状态切换——状态圆点颜色变化伴随 200ms 缩放脉冲。所有动画遵循 prefers-reduced-motion: reduce 时降级为瞬时切换。动画时长统一用三个档位：fast 150ms / normal 250ms / slow 400ms。
解决什么问题：当前界面无动画反馈，操作感觉「生硬」。微交互让界面有呼吸感，用户能感知到操作已被接收。

Pf 性能体验优化

P14 虚拟滚动 + 渐进式渲染
DataPerf

提示词

请优化大数据量下的渲染性能：当报文数量超过 500 条时启用虚拟滚动——仅渲染可视区域内的表格行（约 20 行），滚动时动态复用 DOM 节点。文件解析采用 Web Worker 后台线程，主线程不阻塞，UI 保持响应。解析进度在底部状态栏显示「已解析: 320 / 1024 (31%)」+ 细长进度条。解析完成后数据分批渲染——每批 50 行，用 requestAnimationFrame 调度避免帧丢失。Hex Dump 视图延迟渲染，仅在用户切换到「原始数据」Tab 时才计算。表格搜索过滤使用 debounce 300ms 避免频繁重渲染。
解决什么问题：大 pcap 文件可能包含上万条报文，一次性渲染会卡死界面。虚拟滚动 + 分批渲染确保始终流畅。

IA 信息架构优化

P15 Tab 重组 + 统计视图新增
LayoutData

提示词

请重新组织信息架构：顶部 Tab 从 5 个重组为 4 个——「概览」（默认，显示全部报文表格）、「类型统计」（饼图 + 柱状图展示各 ICMP 类型占比）、「时间线」（按时间轴排列报文事件，支持缩放）、「高级筛选」（多条件组合筛选面板）。原 Tab 中的「类型/代码/检验和/分类」改为表格列的排序和筛选功能，点击列头排序，列头右侧小图标展开筛选下拉。新增「统计」全局视图入口在顶栏导航中，展示总报文数、类型分布、源 IP Top 10、时间跨度等汇总信息，用 4 个指标卡片 + 2 个图表展示。
解决什么问题：原 Tab 将「类型/代码/检验和」作为并列视图不合理——它们是数据维度而非独立功能。重组后 Tab 更符合用户心智模型。

F 完整重设计

P16 一键生成完整重设计方案（综合提示词）
LayoutVisualInteractionA11y

提示词（综合版，可直接粘贴使用）

请为一个 ICMP 差错报文分析工具（桌面端 PyQt/Electron 应用）设计全新的 UI 界面。要求： 【布局】主从式双栏布局——左侧 240px 固定控制面板（数据源分段选择 + 可折叠参数区 + 底部固定操作按钮 + 状态指示器），右侧自适应主内容区（顶部下划线 Tab 导航 + 中间数据表格 + 底部字段详情联动面板，上下可拖拽分隔条调整比例）。 【视觉】Design Token 系统：主色 #378ADD Blue，语义色 Success #639922 / Warning #BA7517 / Danger #E24B4A / Purple #7F77DD，灰阶 7 级 #F8F9FA→#2C2C2A。系统无衬线字体，字号 10-18px 阶梯，仅 400/500 字重。圆角 6/8/12px，间距 4px 基准，边框 0.5px。 【组件】按钮三层权重（主操作实心 36px / 次操作线框 32px / 工具栏图标 28px）；表单输入统一 30px 高度 + 0.5px 灰色边框 + focus 蓝色环；Tab 下划线式选中 + 灰色未选中；表格 sticky 表头 + 行 hover/选中高亮 + 类型 Badge 颜色编码；空状态三段式（图标+描述+CTA）。 【交互】表格行点击联动详情面板展开；键盘 ↑↓ 导航 + Enter 展开原始数据；拖拽文件上传 + Ctrl 快捷键；所有动画 150-250ms 过渡，支持 prefers-reduced-motion。 【状态】空状态（引导 CTA）/ 加载态（骨架屏+脉冲）/ 错误态（红色提示+重试）/ 完成态（绿色对勾+统计）四种状态全覆盖。 【无障碍】WCAG AA 对比度 ≥ 4.5:1，ARIA 标签全覆盖，键盘 Tab 顺序合理，焦点环 2px 可见。 【主题】亮色/暗色/跟随系统三模式，暗色用 #1A1A18 系列非纯黑，localStorage 记忆偏好。 请输出完整的 HTML/CSS/JS 高保真原型，可直接在浏览器中预览交互。
使用方式：这是综合提示词，融合了 P01-P15 的核心要点。粘贴到 AI 对话中即可生成完整的重设计原型。可根据需要删减不相关的段落。

使用指南

  1. 按需取用：每个提示词独立可用，根据当前痛点选择对应维度的提示词
  2. 组合使用：可串联多个提示词——先用 P03 建 Token，再用 P04 优化按钮，最后用 P13 加动画
  3. 综合方案：需要整体重设计时直接用 P16 综合提示词
  4. 迭代优化：每轮优化后对照检查清单验证——对比度达标？键盘可达？状态完整？动画流畅？
  5. 上下文补充：使用提示词时建议附带当前界面截图，AI 能更精准地定位问题
